Hostrig is an **opinionated self-hosted PaaS on k3s**.

Most apps need a **database**, **cache**, **object storage**, and a **runtime**. Hostrig runs that stack on infrastructure you control: typed **services** in a project, **explicit bindings** for credentials, **user apps under gVisor**, and a **platform wildcard URL** via Traefik plus your edge.

It is **not** Coolify/Dokploy with a new coat of paint. It is not “Docker host + Caddy.” It is not a general Kubernetes control plane.

## Why not Coolify / Dokploy?

Those products win on **breadth**: Compose, templates, many databases, fleets. Hostrig wins on a **narrow Railway-shaped loop**:

| Hostrig | Coolify / Dokploy |
| ------ | ----------------- |
| Typed services + explicit bindings | Resource catalogs and Compose |
| gVisor by default for user apps on k3s | Usually plain Docker / runc |
| Postgres + Redis + platform S3 only | MySQL, Mongo, and more |
| Platform wildcard URLs in v1 | Custom domains / many ingress modes |
| Dashboard + MCP for agents | Broader ops surface |

If you need the kitchen sink, use them. If you want sandboxed **app + Postgres + Redis + S3** with git push on k3s, use us. Full list: [Scope](/docs/concepts/scope/).

## The loop

```text
install control plane (Docker Compose)
  → connect or create k3s cluster (+ gVisor on nodes)
  → Domains: base domain + edge → Traefik
  → Registries: default build registry (for git builds)
  → create empty project
  → add web / worker / postgres / redis as needed
  → bind apps to data (DATABASE_URL / REDIS_URL)
  → deploy (image | Dockerfile | Railpack) under gVisor
  → URL on {project}.{baseDomain}
  → scheduled Postgres backups → platform S3
```

## Honest limits (v1)

- **No custom domains** — every web service is under `*.{baseDomain}` (custom domains are v2)
- **TLS at the edge** — Traefik in-cluster is HTTP; terminate TLS at Cloudflare / NetBird / Tailscale
- **No MicroVMs** — gVisor only; we do not claim unbreakable or Dirty Pipe immortality
- **gVisor is the default, not a slogan** — `DEPLOW_APP_RUNTIME=runc` is a temporary escape hatch
- **MCP does not auto-provision Postgres** — `deploy_from_git` ships a web service from a repo; data services and bindings are separate steps
- **Observe is optional** — Deploy works without ClickHouse; Observability is an addon
